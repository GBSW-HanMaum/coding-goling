# 05. 스트레치 — 프로필 페이지 / 진짜 계정·백엔드

> 시간이 정말 남을 때만. 해커톤 데모는 로컬스토리지 기반 게스트 모드로도 충분히 완결됨.

## 프로필 페이지 (`/profile`, 현재 없음)
- `Sidebar.tsx`의 `NAV_ITEMS`에 프로필 메뉴 추가
- 표시 내용: 선택 언어(`activeCourse`), 스트릭, 총 XP, 젬, 완료 레슨 수
- **취약 개념 리스트**: 지금 `challenge.id`만 완료 여부로 추적하고 있어서, 개념 태그가 없음.
  `Challenge` 타입에 `conceptTags?: string[]` 필드 추가(예: `["loop", "variable"]`),
  콘텐츠(`content.ts`)에 태그 채워두고, 스토어에 오답 기록 시 태그별 카운트 집계하는 로직 추가
  → "반복문에서 자주 틀려요" 같은 문구를 프로필에 노출

## 실제 계정/서버 (지금은 전부 브라우저 로컬 — 기기 바꾸면 진행상황 날아감)
데모 자체엔 필수 아니지만, 발표에서 "확장 가능성"으로 언급하기 좋음.
- 가장 가벼운 옵션: Supabase(Auth+Postgres 한 번에) 붙이기 — Clerk+Neon 따로 세팅하는 것보다 해커톤 시간엔 더 빠름
- Zustand 스토어의 `persist(localStorage)`를 서버 동기화로 바꾸는 작업이라, 온보딩(01)/실행엔진(03)/AI(04)보다 우선순위 낮게

## 실제 리더보드 (지금은 `RIVALS` 고정 8명 목업)
- 서버 없이 데모에서 "그럴듯하게" 보이려면: 같은 브라우저의 여러 세션이 아니라
  그냥 지금처럼 고정 라이벌 유지 + 내 XP만 실시간 반영하는 것도 데모용으론 충분함 (이미 그렇게 되어있음)
- 진짜 여러 사용자 순위가 필요하면 위 Supabase 붙인 후에나 의미 있음 — 백엔드 없이는 손대지 말 것
